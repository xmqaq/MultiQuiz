# 📚 MultiQuiz

**MultiQuiz** 是一款优雅、高效的**纯前端题库管理与刷题解决方案**。它专为希望完全掌控自己数据的用户设计，无需后端服务器，所有数据均安全地存储在用户浏览器本地。

## ✨ 核心特性

- **🗂️ 灵活的科目管理**：支持多科目分类，轻松构建结构化的知识体系。
- **📥 智能题目导入**：强大的 `xlsx` 集成，支持 Excel 和纯文本解析，告别繁琐的手动输入。
- **🎯 深度定制的考试模式**：
  - **动态选题**：支持全量、错题优先、权重随机及收藏夹选题。
  - **限时挑战**：模拟真实考试环境，倒计时自动交卷。
- **📈 学习足迹追踪**：
  - **错题本**：自动捕获薄弱环节，支持针对性强化练习。
  - **可视化统计**：直观展示练习频次与正确率走向。
- **🔒 数据主权归还**：数据仅存在于本地 `LocalStorage`，提供 JSON 备份/恢复功能，实现跨设备迁移。
- **📱 全设备适配**：精心设计的响应式布局，在 PC 端和移动端均能提供流畅的交互体验。

## 🛠️ 技术栈

| 维度         | 技术选型         | 说明                                         |
| ------------ | ---------------- | -------------------------------------------- |
| **核心框架** | **Vue 3**        | 使用 Composition API (`<script setup>`) 开发 |
| **构建工具** | **Vite 6**       | 极致的开发体验与构建速度                     |
| **状态管理** | **Pinia 3**      | 模块化的状态解耦设计                         |
| **类型检查** | **TypeScript**   | 全局类型定义，保障代码健壮性                 |
| **路由导航** | **Vue Router 5** | 丝滑的单页应用切换                           |
| **数据解析** | **xlsx**         | 实现高效的本地文件处理                       |
| **单元测试** | **Vitest**       | 确保业务逻辑（如选题算法）的准确性           |

## 📂 项目结构

项目采用模块化设计，将 UI 渲染与核心业务逻辑（Services）完全分离：

```bash
src/
├── components/       # 视图组件 (通用、布局、页面级)
├── services/         # 核心逻辑 (存储、导入解析、考试算法、数据迁移)
├── stores/           # Pinia 状态树 (题库、考试会话、UI 状态)
├── types/            # 强类型定义中心
├── router/           # 路由导航配置
├── tests/            # 单元测试用例
└── App.vue           # 根入口
```

## 🚀 快速开始

### 1. 环境准备

确保你的环境中已安装 [Node.js](https://nodejs.org/) (建议 v18+)。

### 2. 安装与启动

```bash
# 克隆项目
git clone https://github.com/xmqaq/MultiQuiz.git

# 进入目录
cd MultiQuiz

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 3. 构建生产版本

```bash
npm run build
```
