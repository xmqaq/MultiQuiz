# MultiQuiz

一个面向个人复习与多学科题库管理的前端练习平台。  
支持题库导入、题目浏览、模拟答题、错题练习、收藏与标签管理、考试记录追踪，以及基于本地数据的统计分析。

MultiQuiz 采用纯前端实现，默认使用浏览器 `localStorage` 持久化数据，无需后端、无需数据库，打开即可使用，适合本地学习、自建静态部署和个人题库长期维护。

## 项目特色

- 多学科题库统一管理，适合课程复习、考试刷题和知识点分科维护
- 支持 Excel / JSON 导入，内置模板文件，降低题库整理成本
- 提供题目浏览、搜索、筛选、收藏、标签编辑等精细化管理能力
- 支持模拟考试、错题练习、重做此卷、未完成会话恢复
- 提供考试记录、学习热力图、正确率趋势、学科雷达图等统计视图
- 支持完整备份与恢复，题库、错题、标签、收藏、统计和历史记录可整体迁移
- 纯静态架构，部署简单，适合本地运行或挂载到任意静态站点

## 适用场景

- 课程复习与期末备考
- 政治、历史、计算机基础等单选题题库练习
- 多门学科并行刷题
- 个人长期积累错题与重点标签
- 无后端条件下的轻量自用题库系统

## 核心功能

### 1. 题库中心

- 统一查看所有学科题库
- 展示每个题库的题目数、错题数、收藏数
- 支持单个题库练习、浏览、导出与管理
- 支持题库删除、重命名、去重、整体清空等管理操作

### 2. 导入题库

- 支持 `Excel (.xlsx / .xls)` 与 `JSON` 导入
- 内置模板：
  - [templates/multiquiz-template.xlsx](./templates/multiquiz-template.xlsx)
  - [templates/multiquiz-template.json](./templates/multiquiz-template.json)
- 首次导入可直接基于模板填充题目，减少字段错误
- 支持学科覆盖导入和题库恢复导入

### 3. 题目浏览

- 支持按学科、关键词、标签、收藏状态筛选
- 支持显示/隐藏答案模式
- 支持题目收藏
- 支持标签管理与快捷筛选
- 支持将当前范围题目导出为 JSON / 文本 / PDF（打印）

### 4. 模拟答题

- 支持单学科或全部学科混合练习
- 支持限定题量或整库练习
- 支持限时答题
- 支持错题优先、加权抽题、优先“需复习”标签、仅练收藏题等智能策略
- 支持答题卡跳转、进度展示、提交评分与结果反馈
- 支持未完成考试自动恢复

### 5. 错题本

- 自动沉淀答错题目
- 支持按学科过滤错题
- 支持错题再次练习
- 支持清空错题与状态同步更新

### 6. 考试记录

- 自动记录每次正式模拟的时间、分数、正确数、错题数、耗时
- 支持查看详情
- 支持删除单条记录
- 支持导出历史记录
- 支持“重做此卷”快速复练

### 7. 统计分析

- 学习数据总览
- 学习热力图
- 学科正确率雷达图
- 练习趋势
- 正确率趋势

## 技术栈

- `HTML5`
- `CSS3`
- `Vanilla JavaScript`
- [SheetJS / xlsx](https://cdnjs.com/libraries/xlsx) 用于 Excel 解析
- 浏览器 `localStorage` 用于本地数据持久化

## 快速开始

### 方式一：直接打开

1. 下载或克隆项目到本地
2. 直接打开 [index.html](./index.html)
3. 开始导入题库并使用

### 方式二：使用本地静态服务

如果你希望以更接近部署环境的方式运行，可以在项目目录执行：

```bash
python3 -m http.server 8000
```

然后在浏览器访问：

```text
http://localhost:8000
```

## 题库格式说明

### JSON 格式

支持以下两种结构：

```json
[
  {
    "question": "中国近代史的开端是？",
    "optionA": "鸦片战争",
    "optionB": "甲午战争",
    "optionC": "辛亥革命",
    "optionD": "五四运动",
    "answer": "A"
  }
]
```

或：

```json
{
  "questions": [
    {
      "question": "中国近代史的开端是？",
      "optionA": "鸦片战争",
      "optionB": "甲午战争",
      "optionC": "辛亥革命",
      "optionD": "五四运动",
      "answer": "A"
    }
  ]
}
```

### Excel 格式

Excel 模板中第一行为表头，字段顺序固定为：

```text
题目 | 选项A | 选项B | 选项C | 选项D | 正确答案
```

要求：

- 每道题必须包含题干与 `A / B / C / D` 四个选项
- `answer` 仅支持 `A / B / C / D`
- 建议直接使用项目内模板填写，兼容性最好

## 数据存储与备份

项目默认将数据保存在浏览器本地，不依赖远程服务。

本地保存内容包括：

- 学科与题目数据
- 错题本
- 考试记录
- 练习统计
- 标签数据
- 收藏状态
- 练习日志

你可以通过界面中的“导入配置 / 导出备份”功能完成整库迁移与恢复。

备份文件为 `JSON`，适合：

- 更换浏览器或设备时迁移数据
- 大幅修改题库前先做快照
- 本地数据清理前保留完整副本

## 离线与兼容性说明

- 项目主体为纯前端，本地即可运行
- Excel 导入依赖页面中加载的 `xlsx` CDN 脚本
- 如果外部脚本加载失败，系统会自动降级为仅支持 `JSON` 导入
- 推荐使用现代浏览器：
  - Chrome
  - Edge
  - Safari
  - Firefox

## 设计原则

- 纯静态、低门槛部署
- 本地优先，数据由用户掌控
- 面向多学科练习，而不是单一题库页面
- 功能完整，但尽量保持使用成本低
- 兼顾桌面端与移动端体验

## 适合后续扩展的方向

- 多题型支持，如判断题、填空题、多选题
- 账号同步或云端备份
- 更细粒度的学习报告
- 题库分享与协作导入
- PWA 离线缓存与安装能力

## 部署建议

由于项目没有后端依赖，你可以直接部署到任意静态托管平台，例如：

- GitHub Pages
- Vercel
- Netlify
- Nginx 静态站点

部署时只需保留静态文件与 `templates` 目录即可。
