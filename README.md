# MultiQuiz

一个面向个人复习与多学科题库管理的前端练习平台。  
支持题库导入、题目浏览、模拟答题、错题练习、收藏与标签管理、考试记录追踪，以及基于本地数据的统计分析。

MultiQuiz 采用纯前端实现，默认使用浏览器 `localStorage` 持久化数据，无需后端、无需数据库，打开即可使用，适合本地学习、自建静态部署和个人题库长期维护。

## 项目特色

- 多学科题库统一管理，适合课程复习、考试刷题和知识点分科维护
- 支持 Excel / JSON 导入，内置模板文件，降低题库整理成本
- 提供题目浏览、搜索、筛选、收藏、标签编辑等精细化管理能力
- 支持模拟考试、错题练习、重做此卷、未完成会话恢复
- 提供考试记录、学习热力图、正确率趋势、学科雷达图等统计视图
- 支持完整备份与恢复，题库、错题、标签、收藏、统计和历史记录可整体迁移
- 纯静态架构，部署简单，适合本地运行或挂载到任意静态站点

## 适用场景

- 课程复习与期末备考
- 政治、历史、计算机基础等单选题题库练习
- 多门学科并行刷题
- 个人长期积累错题与重点标签
- 无后端条件下的轻量自用题库系统

## 核心功能

### 1. 题库中心

- 统一查看所有学科题库
- 展示每个题库的题目数、错题数、收藏数
- 支持单个题库练习、浏览、导出与管理
- 支持题库删除、重命名、去重、整体清空等管理操作

### 2. 导入题库

- 支持 `Excel (.xlsx / .xls)` 与 `JSON` 导入
- 内置模板：
  - [templates/multiquiz-template.xlsx](./templates/multiquiz-template.xlsx)
  - [templates/multiquiz-template.json](./templates/multiquiz-template.json)
- 首次导入可直接基于模板填充题目，减少字段错误
- 支持学科覆盖导入和题库恢复导入

### 3. 题目浏览

- 支持按学科、关键词、标签、收藏状态筛选
- 支持显示/隐藏答案模式
- 支持题目收藏
- 支持标签管理与快捷筛选
- 支持将当前范围题目导出为 JSON / 文本 / PDF（打印）

### 4. 模拟答题

- 支持单学科或全部学科混合练习
- 支持限定题量或整库练习
- 支持限时答题
- 支持错题优先、加权抽题、优先“需复习”标签、仅练收藏题等智能策略
- 支持答题卡跳转、进度展示、提交评分与结果反馈
- 支持未完成考试自动恢复

### 5. 错题本

- 自动沉淀答错题目
- 支持按学科过滤错题
- 支持错题再次练习
- 支持清空错题与状态同步更新

### 6. 考试记录

- 自动记录每次正式模拟的时间、分数、正确数、错题数、耗时
- 支持查看详情
- 支持删除单条记录
- 支持导出历史记录
- 支持“重做此卷”快速复练

### 7. 统计分析

- 学习数据总览
- 学习热力图
- 学科正确率雷达图
- 练习趋势
- 正确率趋势

## 快速开始

### 方式一：直接打开

1. 下载或克隆项目到本地
2. 直接打开 [index.html](./index.html)
3. 开始导入题库并使用

### 方式二：使用本地静态服务

如果你希望以更接近部署环境的方式运行，可以在项目目录执行：

```bash
python3 -m http.server 8000
```

然后在浏览器访问：

```text
http://localhost:8000
```

## 使用流程

1. 进入“导入题库”，填写学科名称并选择 Excel / JSON 文件。
2. 在“题库中心”查看题库卡片，可快速进入练习或浏览。
3. 在“题目浏览”中用关键词、标签、收藏状态筛选题目。
4. 在“模拟答题”中选择学科、题量、时间和智能抽题策略后开始练习。
5. 提交答卷后，系统会自动更新错题本、考试记录和统计数据。
6. 定期通过“导出备份”保存完整数据，迁移设备时用“导入配置”恢复。

## 题库格式说明

每道题目前按单选题处理，题干、四个选项和答案都是必填字段。

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| `question` | 题干 | `中国近代史的开端是？` |
| `optionA` | A 选项 | `鸦片战争` |
| `optionB` | B 选项 | `甲午战争` |
| `optionC` | C 选项 | `辛亥革命` |
| `optionD` | D 选项 | `五四运动` |
| `answer` | 正确答案，仅支持 `A / B / C / D` | `A` |

### JSON 格式

支持以下两种结构：

```json
[
  {
    "question": "中国近代史的开端是？",
    "optionA": "鸦片战争",
    "optionB": "甲午战争",
    "optionC": "辛亥革命",
    "optionD": "五四运动",
    "answer": "A"
  }
]
```

或：

```json
{
  "questions": [
    {
      "question": "中国近代史的开端是？",
      "optionA": "鸦片战争",
      "optionB": "甲午战争",
      "optionC": "辛亥革命",
      "optionD": "五四运动",
      "answer": "A"
    }
  ]
}
```

### Excel 格式

Excel 模板中第一行为表头，字段顺序固定为：

```text
题目 | 选项A | 选项B | 选项C | 选项D | 正确答案
```

要求：

- 每道题必须包含题干与 `A / B / C / D` 四个选项
- `answer` 仅支持 `A / B / C / D`
- 建议直接使用项目内模板填写，兼容性最好
- Excel 导入依赖页面加载的 `xlsx` 脚本；加载失败时页面会自动降级为 JSON 导入

## 数据存储与备份

项目默认将数据保存在浏览器本地，不依赖远程服务。

本地保存内容包括：

- 学科与题目数据
- 错题本
- 考试记录
- 练习统计
- 标签数据
- 收藏状态
- 练习日志
- 未完成考试会话

你可以通过界面中的“导入配置 / 导出备份”功能完成整库迁移与恢复。

备份文件为 `JSON`，适合：

- 更换浏览器或设备时迁移数据
- 大幅修改题库前先做快照
- 本地数据清理前保留完整副本

## 离线与兼容性说明

- 项目主体为纯前端，本地即可运行
- Excel 导入依赖页面中加载的 `xlsx` CDN 脚本
- 如果外部脚本加载失败，系统会自动降级为仅支持 `JSON` 导入
- 不同浏览器的 `localStorage` 彼此隔离，换浏览器前建议先导出备份
- 浏览器清站点数据会清除题库、错题、记录和设置，请谨慎操作
- 推荐使用现代浏览器：
  - Chrome
  - Edge
  - Safari
  - Firefox

## 开发与维护说明

### 脚本加载顺序

项目仍使用普通浏览器脚本而不是 ES Modules，主要是为了兼容现有 HTML 中的内联事件调用。加载顺序由 [index.html](./index.html) 底部维护：

1. `js/core/`、`js/state/`、`js/ui/` 先加载基础能力
2. `js/render/` 和 `js/modules/` 加载页面渲染与业务逻辑
3. `js/app.js` 定义 `initApp()` 和总装配逻辑
4. `script.js` 作为唯一启动入口，在 DOM 就绪后调用 `initApp()`

不要直接删除 `script.js` 或把启动逻辑分散到多个模块里。它虽然很短，但负责保持入口清晰，也符合当前项目结构。

### 模块职责

| 目录 | 职责 |
| --- | --- |
| `js/core/` | 常量、工具函数、导入题校验 |
| `js/state/` | 全局状态、持久化、历史数据修复 |
| `js/modules/` | 题库、导入、浏览、考试、错题、历史、统计、备份等业务逻辑 |
| `js/render/` | 各页面和图表的 DOM 渲染 |
| `js/ui/` | 弹窗、Toast、侧边栏、静态事件绑定、响应式行为 |

### 本地检查

修改 JS 后建议至少做两类检查：

```bash
for f in script.js js/**/*.js; do node --check "$f" >/dev/null || exit 1; done
```

```bash
python3 -m http.server 8000
```

然后访问 `http://localhost:8000`，确认导入、浏览、答题、错题本、历史记录和统计页能正常使用。

## 设计原则

- 纯静态、低门槛部署
- 本地优先，数据由用户掌控
- 面向多学科练习，而不是单一题库页面
- 功能完整，但尽量保持使用成本低
- 兼顾桌面端与移动端体验

## 适合后续扩展的方向

- 多题型支持，如判断题、填空题、多选题
- 账号同步或云端备份
- 更细粒度的学习报告
- 题库分享与协作导入
- PWA 离线缓存与安装能力

## 部署建议

由于项目没有后端依赖，你可以直接部署到任意静态托管平台，例如：

- GitHub Pages
- Vercel
- Netlify
- Nginx 静态站点

部署时只需保留静态文件与 `templates` 目录即可。
